Antes de modificar código, inspeccionar el proyecto existente.

Codex debe revisar primero:

- estructura de carpetas
- stack técnico
- dependencias
- scripts disponibles
- convenciones de código
- estilo visual existente
- README o documentación del proyecto

No asumir arquitectura, framework, librerías, comandos ni patrones sin revisar primero los archivos reales del repositorio.
